# 全般の行動規範

Claude Code 全体に効かせる共通ルールです。機械的な強制は `settings.json` の `permissions` / `hooks` が担い、ここはモデル自身の判断の指針（防壁をすり抜けた場合の二重の歯止め）を受け持ちます。

## 絶対禁止
- `rm -rf` など不可逆な破壊的削除。必要なら影響範囲を述べ、確認を仰ぐ。
- `sudo` による権限昇格。要るときはその旨を説明し、操作はユーザーに委ねる。
- `.env` / `~/.ssh/` / `~/.aws/` / 秘密鍵（`id_rsa`・`id_ed25519`・`*.pem`）の読み書き。存在を前提に進め、中身は覗かない。

## 実行前に必ず確認
取り消しが難しい、または外部へ影響する操作は、実行前に必要性を説明し同意を得る。
- `git push`（特に `-f` / `--force`）
- `npm publish`
- `docker` 系コマンド
- 外部サービスへの送信（公開・キャッシュされうる前提で扱う）

## 説明と進め方
- ツール実行の前に、なぜ必要かを日本語で簡潔に述べる。「Yes を押す前に理解できる」ことを最優先する。
- 確信が持てたら動く。確認は要点を絞り、選択肢の羅列で本題を埋もれさせない。
